Assignment

 

More users - Find Places that Need Testing

Based on the card sort and user scenarios you've done you should do the following:

1) make a list of navigation questions you have. Examples:

  • should you have a “for students”, and a “for faculty” section
  • should you have “tuition” in the main navigation
  • should you list all the programs that the College offers?
  • Should you make a distinction between undergraduate and graduate classes?

2) highlight any wording you're not sure about? Examples:

  • About Us / About / History
  • Financial Aid / Costs / Tuition
  • Make a list of content questions you have on home page. For example: should you show the latest news on the home page?
  • Make a list of questions that your key users might have when coming to the site.

Card Sort Review

I have produced another card sort. The original probably had too many top level categories for the home page, making that page hard for anyone to quickly spot the route to their goal. The new card sort will probably make the home page easier to read, and quicker to use for some, but will probably hinder other users.

Essentially, the design becomes a compromise. I need to keep in mind the following keys:

  • University Mission Statement.

  • Web Site Users and their Goals.

  • What works for the Client.

The User Groups are:

  • Prospective Student

  • Current Student

  • Parents

  • Alumni

  • Faculty

  • Media

  • Researchers

  • Corporate

  • Staff

  • Local community

  • Job Seeker

  • Curious Users

For a full listing of user groups and their goals, click here.

The search engine user group has been dropped from the list for the moment, as I do not have enough information regarding this user group.

The compromise needs to maximally facilitate the keys. Facilitating the keys and the compromised appears to be a considerable challenge.

Navigation Questions

I plan to use the Design 2 card sort as the basis for the project. With far fewer links, I feel this design is more expandable than the original and the modified original. If the usability testing reveals the need for additional links, I plan to alter the card sort appropriately.

Navigation Questions:

I really don't know how far to go with this. I would think the whole card sort needs to be examined in detail, but this exercise seems to be limited to just the home page. I thus limit my attentions to issues regarding the home page.

The keys here are to provide easy navigation to the user groups that no longer have an obvious link on the home page, and to ensure that all user groups can intuitively navigate the home page. Does the user understand how the site is organized, how it works? Do the keywords make sense?

The aim of the wireframe exercise is to look for what is missing, for what is not clear, sollicit suggested improvements, remove navigation problems before they become embedded into the design of the page.

All answers should be followed by the why question, when appropriate.

  • What sort of organization do you think this home page is for? What is the purpose of the site?

  • For each of my key user groups, verify that the link for them is obvious.

  • Would a parent choose Prospective Students? Do parents need their own nav?

  • Should Staff/Faculty be separate categories?

  • Do I need a Help item on the home page? If a user cannot find what he is looking for, what would he do next? Why? Why could he not find what he was looking for? What did I miss?
  • A media user would need to start with The University. Is this obvious? Where should this link be. Why?
  • A Job Seeker user would need to start with The University. Is this obvious? Where should this link be? Why?
  • A local community user would need to start with Quick Links. Is this obvious? Where should this link be? Why?
  • What information would each user group expect to find in the Quick Links? Why?
  • Need to ensure that all of the prospective student types know to start with the aptly named link. Is it obvious? Why?
  • Where would a High School Teacher, researching UG courses, navigate to? Why? Any suggestions?
  • Would a high school student intuitively understand the term prospective student? Is there another term? Why?
  • What would you do and why would you do it? How could it be improved?

Wording Questions:

Look for words that do not make sense, or which could be improved. Words need to be obvious, the user should instantly recognize the words and be able to instantly act on them.

  • Is it Prospective or Future Students?

  • Should I use Graduates or Alumni or Both? If both, Graduates/Alumni? What? Why?

  • Should I use Staff or Faculty or Both? If both, Staff/Faculty, Faculty/Staff? What? Why?
  • Is corporate the right word? Maybe Business would be better.
  • Is Quick Links the right term? What does that mean to the user? What word(s) would the user expect to see? Why?
  • Should it be The University, or About The University, or About Us, or what?
  • Should I show the latest news on the home page?
  • Should it be media or press?
  • Should it be Home? Or The University? Or what? Why?
  • Local community or Neighbors?
