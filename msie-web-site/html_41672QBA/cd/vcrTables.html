Status

 

This is a worker table used primarily to ensure consistent status usage across the system.

Values of status are:

1 = New, 2 = Approved, 3 = Confirmed, 4 = Offer, 5 = Reject, 6 = Inventory, 7 = Accept, 8 = Invoiced, 9 = Consigned

It is better for the program to key of an integer than a String. This design also forces the user interface developer to get the string from this table, thus helping to localize the product.

Column Name Type Non-Unique Comments
Id integer unique  
status String
