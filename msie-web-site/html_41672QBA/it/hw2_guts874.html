SunONEApplication Server configuration for CGIs under Solaris 9

 

Create a new user jvcgi with a new group jvcgi and a new home directory /export/home/jvcgi

Create a new directory cgi-bin under the docroot directory.

chown jvcgi:jvcgi cgi-bin

 

The aim is to have the CGI files owned by this user and run by this user, the CGI files will only be run from the $DOCROOT/cgi-bin directory. Do not put anything else in this directory, no non cgi files, no sub directories.

 

cp *.cgi $DOCROOT/cgi-bin

cd $DOCROOT/cgi-bin

chown jvcgi:jvcgi *.cgi

chmod 755 *.cgi

 

The files are ready, we just need to tell the Application Server what to do.

 

From the Sun, as the free version of the Application Server does not support remote administration, open a browser from the Sun and:

http://localhost:4848

 

This begins the Application Server Administrator. Give the administrator user and password to enter.

Choose the domain that requires configuration, example S1.

Select the Virtual Servers Page.

Select the Virtual Server

Click on the General Tab

Set the User field to jvcgi

Set the group field to jvcgi

Set the cgi directory to $DOCROOT/cgi-bin (or in this example /jvbin/appserver_dir/sunONEdomain/S1/docroot/cgi-bin

Click on the Save button

Go to the server instance page and select the apply button.

Restart the application server.

 

To test: http://jvsun:4849/count.html where count.html is:

 

<html>
<body>
You fred are visitor number
<img src=cgi-bin/count.cgi>
ending here
</body>
</html>

The count.cgi script is a site visitor counter, a little primitive but seemingly rather effective.
